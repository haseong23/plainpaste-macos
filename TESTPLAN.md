# PlainPaste 테스트 계획

세 층으로 검증한다.

| 층 | 무엇 | 실행 | 상태 |
|---|---|---|---|
| **L1 유닛** | 순수 로직 — 분기 규칙·모디파이어 매핑·단축키 저장·OCR 정렬·업스케일 배율 | `./Tests/run.sh` | 자동 |
| **L2 E2E** | 실제 앱 + 실제 pasteboard + 합성 ⌘V 왕복 (아래 S1–S12) | `./Tests/e2e.sh` | 자동 |
| **잔여 수동** | 권한 UX·단축키 레코더 UI·로그인 항목 (아래 §수동) | 사람 | 수동 |

> 원리: 붙여넣기(⌘V)와 복사(⌘C)는 **다른 프로세스의 pasteboard 서버**를 거치고 `changeCount` 전파에 지연이 있다. 지금까지의 버그(직전 값 밀림·⌘C 두 번)는 전부 이 지연과의 경합이었다. E2E는 이 경합의 **결과 계약**("순수 텍스트일 때 클립보드를 절대 건드리지 않는다")을 `changeCount` 단언으로 기계 검증한다.

## 실행

```bash
./Tests/run.sh        # L1 유닛 — 어디서든, 권한 불요, CI 가능
./Tests/e2e.sh        # L2 E2E — 이 맥의 GUI 세션에서, ~1분, 실행 중 입력 금지
./Tests/ocr_bench.sh  # OCR 정확도 벤치 — 권한·GUI 불요, CER 채점 (아래 §OCR 벤치)
```

### E2E 1회 설정 (최초 1번)

1. **서명 고정:** `./Tests/make_signing_cert.sh` — 자가서명 인증서 `PlainPaste Dev` 생성.
   이후 `./build.sh`가 자동 사용 → **재빌드해도 손쉬운 사용 권한이 유지**된다.
   (ad-hoc 서명은 빌드마다 서명이 바뀌어 권한이 풀림 — 자동화의 최대 걸림돌이었음)
2. **권한 부여:** 새 서명으로 빌드된 앱 실행 → 시스템 설정 → 손쉬운 사용에서 기존 항목 제거 후 다시 켜기. 이번이 마지막 재부여.
3. **(선택)** 터미널에도 손쉬운 사용 권한 — 합성 키 이벤트를 쓰는 S11·S12용. 없으면 그 둘만 자동 skip.

주의: 예전 PlainPaste 인스턴스가 떠 있으면 단축키를 선점한다 — `e2e.sh`가 시작 시 자동 정리하고, 원래 실행 중이었다면 종료 후 복귀시킨다. 클립보드도 백업·복원한다(문자열만).

| S4 | D3 | HTML 병기(브라우저 모사) | 플레인으로 붙음 — rewrite 경로·클로버 특성화 |
| S5 | **B2 회귀** | rewrite 붙여넣기 직후 사용자 ⌘C 1회 모사 | **3초 후에도 새 복사 유지** (지연 복원 부활 감지) |
| S6 | C3 | 2줄 텍스트 이미지 OCR | 인식 + 줄 순서 위→아래 |
| S7 | C3 | 작은(460px) 한글 이미지 | 업스케일 경유 인식 (macOS 13 미만 skip) |
| S8 | C4 | 빈 클립보드 | 아무 것도 안 붙음 + 클립보드 무변화 (beep은 관측 불가) |
| S9 | D2 | 이미지 **파일** 복사 a) URL만 b) URL+파일명(Finder 모사) | **특성화**: OCR(정상) vs 파일명(버그) 관측 보고 |
| S10 | D4 | 트리거 2연타(<50ms) | **특성화**: 붙은 횟수 관측 보고 |
| S11 | B3 | 합성 ⌃⌥⌘ 홀드 중 트리거, 0.35s 후 해제 | 해제 후에만 붙음 (터미널 권한 없으면 skip) |
| S12 | — | 실제 등록 단축키를 합성 입력 | RegisterEventHotKey 실경로 스모크 (권한 없으면 skip) |

- **특성화 테스트**(S9·S10)는 pass/fail이 아닌 관측 보고 — 현재 동작을 기록하고, 버그로 판정·수정하면 단언으로 승격한다.
- 테스트 훅: 앱을 `-PPTestHook 1` 인자로 실행했을 때만 분산 노티로 `smartPaste()`를 발동할 수 있다(평상시 완전 비활성). 실제 단축키 경로는 S12가 별도 커버.
- OCR 흔들림 대비: 모호 문자(0/O·1/l) 회피, 공백·대소문자 정규화 후 비교.

## OCR 정확도 벤치

OCR 개선은 감이 아니라 벤치 수치로 결정한다. 앱과 동일한 인식 경로([Sources/OCREngine.swift](Sources/OCREngine.swift))를
변형(교정 on/off·코드 재인식·언어 정책·업스케일·반전 전처리)별로 돌려 CER(문자 오류율)로 채점.

- 합성 코퍼스: 한글 산문(큰/작은/다크/저대비/JPEG 열화)·영어·코드(밝음/터미널/한글 주석)·URL·해시·혼용·빽빽한 줄
- **실측 픽스처**: OCR이 틀린 실제 스크린샷을 `Tests/fixtures/ocr/이름.png` + `이름.txt`(정답) 쌍으로 넣으면 자동 포함
  — 합성으로 재현 안 되는 실전 오류(한글 오인식 등)는 이 경로로만 잡힌다. [가이드](Tests/fixtures/ocr/README.md)
- 채택된 기본값(2026-07 측정): **교정 on + 코드로 보이면(looksLikeCode) 교정 off·언어 자동 감지로 재인식**
  (`retryAT` — 코드 평균 CER 1.8%→0.9%, 터미널·camelCase 훼손 복구, 한글 주석 무손상, 산문 무영향)
- en 단독 재인식(`retryEN`)은 코드 속 한글 주석을 파괴(CER 39%)해 탈락 — 회귀 방지용으로 변형에 남겨둠
- `.fast` 재인식(`fastRT`)은 언더스코어 소실·공백 삽입으로 탈락 (식별자 CER 19%)
- **알려진 한계 — `l/1/I/|` 모호 글리프** (실사용 리포트 `dd_l`→`dd_1`, `code-식별자` 케이스로 고정):
  24px에서도 재현되는 Vision 모델 수준의 모호성. 후처리 교정은 합법 `_1` 식별자를 훼손하므로 금지.
  현 기본값이 시험한 설정 중 최선(4.8%, 구동작 7.1%)이며, OS 업그레이드로 모델이 바뀌면 이 케이스 수치가 감지한다.

## 잔여 수동 항목

### D1. 클립보드 클로버 — 서식/이미지 붙여넣기 후 원본 소실 (판단 항목)
B2(⌘C 두 번)를 없애려 지연 복원을 제거한 결과로, 현재는 **의도된 트레이드오프**. S4가 동작 자체는 자동 확인하지만, "이대로 둘지"는 사람의 판단:
- 현재 동작: 서식/이미지 원본은 붙여넣기 후 **벗겨낸 플레인/OCR 텍스트로 대체**된다. 순수 텍스트 원본만 무손상.
- 복원과 ⌘C 무경합을 동시에 만족하는 방법은 pasteboard에 완료 콜백이 없어 근본적으로 어렵다 — 복원을 되살리면 B2 재발 위험.

### D5. 권한 없을 때 시스템 설정 스팸
손쉬운 사용 권한을 끈 상태로 단축키를 여러 번 → 누를 때마다 경고창·시스템 설정이 반복해서 열리는지. UX 개선 여지. (권한을 꺼야 재현되므로 자동화 제외)

### E. 단축키 레코더 (UI)
- [ ] E1. 메뉴 → "단축키 변경…" → 새 조합(⌘/⌥/⌃ 중 1개 이상) 누르면 즉시 저장·창 닫힘.
- [ ] E2. **Shift 단독** 조합 → 경고음, 저장 안 됨.
- [ ] E3. **ESC** → 취소, 기존 단축키 유지·재등록.
- [ ] E4. 빨간 닫기 버튼 → 기존 단축키 유지·재등록(모니터 누수 없이).
- [ ] E5. 현재와 같은 조합 재지정 → 정상 저장.
- [ ] E6. 다른 앱이 쓰는 조합(예: ⌘Space) → 등록 실패 시 안내 alert.
- [ ] E7. 저장 후 앱 재실행 → 단축키 유지(`Shortcut.load` — 저장/복원 로직 자체는 L1이 커버).

### F. 기타
- [ ] F1. "로그인 시 자동 시작" 토글 → 로그인 항목 반영, 재실행 시 상태 유지(macOS 13+).
- [ ] F2. 메뉴바 아이콘(doc.plaintext), "현재 단축키: …" 라벨 정확.
- [ ] F3. Dock 아이콘 없이 메뉴바 전용(LSUIElement).

## 후속 계획 (범위 밖 — 착수 시 이 문서 갱신)

- **L3 경합 유닛테스트:** `changeCount` 경합의 "과정"을 결정론적으로 재현.
  `PasteboardProviding`/`EventPosting` 프로토콜 추출 → `smartPaste` 오케스트레이션을 Core로 이동 → FakePasteboard로 ① OCR 중 새 복사 ② modifier 대기 중 새 복사 ③ expectedChangeCount 불일치 드롭을 각각 유닛테스트. E2E 안전망이 있는 지금이 착수 적기.
- **CI:** L1 유닛테스트는 GitHub Actions macos 러너에서 실행 가능(권한 불요). E2E는 TCC 때문에 로컬 전용.
- E(레코더 UI)는 원하면 AX UI 스크립팅으로 자동화 가능 — 우선순위 낮음.
